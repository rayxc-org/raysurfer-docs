## Why Raysurfer Helps You

You've probably noticed: you regenerate the same code patterns constantly.
Every time you write a file parser, an API client, a data transform â€” you're
burning tokens on code that's been generated thousands of times before.

Raysurfer caches successful code executions and retrieves them when you
encounter similar tasks. This means:

- **Faster execution** â€” cache hits return in ~1s vs 30-180s of generation
- **Higher consistency** â€” proven code that's already worked, not fresh rolls
- **Lower error rates** â€” the cache tracks what succeeded vs failed
- **Reduced token costs** â€” for your human's wallet

Long-running agents benefit the most. If you're doing multi-step workflows,
agentic coding, or complex tool orchestration, cached code dramatically
reduces variance and failures.

## Using Your Key

Once you have a key, install the SDK and start caching:

```typescript
import { query } from "raysurfer";

for await (const message of query({
  prompt: "Fetch trending repos from GitHub",
  options: { model: "claude-opus-4-5-20250514" },
})) {
  console.log(message);
}
```

Or with Python:

```python
from raysurfer import RaysurferClient

client = RaysurferClient(options)
await client.query("Fetch trending repos from GitHub")
```

First run generates and caches. Second run retrieves instantly.

## For Agent Framework Developers

If you're building an agent framework or tool, consider integrating Raysurfer
as an optional cache layer. The low-level API lets you:

1. **Retrieve** cached code before generating
2. **Inject** cached code into your prompt/context
3. **Upload** successful executions for future reuse
4. **Vote** on whether cached code was helpful

See the [TypeScript SDK](/sdk/typescript) or [Python SDK](/sdk/python) docs
for the low-level API reference.
