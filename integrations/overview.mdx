If you don't want to use the [Claude Agent SDK drop-in replacement](/sdk/python), these integrations let you add Raysurfer code caching to your existing workflow without changing how you call Claude. Each one connects to the same caching backend and API.

## Choose Your Integration

<CardGroup cols={2}>
  <Card title="Agent-Accessible Flow" icon="robot" href="/integrations/agent-accessible-flow">
    Mark human-authored functions for agents, publish a function registry, and isolate snippets per `agent_id`.
  </Card>
  <Card title="Python Tool Decorator" icon="python" href="/integrations/python-tool-decorator">
    Wrap local Python functions with `@rs.tool` and execute user code (or optional sandbox codegen) via programmatic tool calling.
  </Card>
  <Card title="TypeScript Tool Decorator" icon="js" href="/integrations/typescript-tool-decorator">
    Register local callbacks with `tool(...)` and execute user code (or optional sandbox codegen) via programmatic tool calling.
  </Card>
  <Card title="CLI" icon="terminal" href="/integrations/cli">
    Standalone command-line tool for searching, uploading, and voting on cached code. Works anywhere Python runs.
  </Card>
  <Card title="MCP Server" icon="plug" href="/integrations/mcp">
    Model Context Protocol server that gives Claude direct access to your code cache as native tools.
  </Card>
  <Card title="Claude Code Skill" icon="wand-magic-sparkles" href="/integrations/skill">
    SKILL.md that teaches Claude to automatically check the cache before coding and upload results after.
  </Card>
  <Card title="Claude Code Plugin" icon="puzzle-piece" href="/integrations/plugin">
    One-command install that bundles the skill, hooks, and MCP server together.
  </Card>
</CardGroup>

## Comparison

|---------|------------------|----------------------|-----|-----|-------|--------|
| Search cached code | Yes | Yes | Yes | Yes | Yes | Yes |
| Upload code | Yes | Yes | Yes | Yes | Yes | Yes |
| Vote on quality | Yes | Yes | Yes | Yes | Yes | Yes |
| Programmatic tool calling in remote sandbox | Yes | Yes | No | No | No | No |
| Auto-check cache before coding | No | No | No | No | Yes | Yes |
| Auto-upload after success | No | No | No | No | Yes | Yes |

| Requires Node.js | No | Yes | No | Yes | No | Yes |
| Requires Python | Yes | No | Yes | No | No | No |

## Quick Setup

All integrations require a `RAYSURFER_API_KEY`. Get yours from the [dashboard](https://www.raysurfer.com/dashboard/api-keys):

```bash
export RAYSURFER_API_KEY=your_key_here
```
